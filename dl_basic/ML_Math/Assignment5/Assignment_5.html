Task 3: Programming example with VC-Dimension

In this task, a programming example should further strenghen your intuition about how model complexity affects the performance of a classifier and how this is related to its VC-dimension.

Task:

First of all we want to create binary labelled data within the unit circle. Therefore implement a function "create_data" that depends on a parameter $k\in \mathbb{N} \geq 2$, which returns $k$-labeled data drawn uniformly from the unit circle with binary labels contained in $\{-1,1\}$, such that the points in one half of the circle should be labeled with $1$, and points in the other half by $-1$. We provide the code for the first two data points, which we label with $+1$ and $-1$, respectively. (The reason for this is that we need to guarantee at least two different labels for the upcoming tasks. Keep in mind that we label them randomly, no matter where they lie in the plane. For creating further points you should proceed differently!) You should now finish the code according to the description above. Hint: to understand the procedure for the first two points, recall that points within the unit circle can be written as $\left( r \cos\varphi, r \sin\varphi\right)$, where $0\le r \le 1$ and $0 \le \varphi \le 2 \pi$.

Next, we want to implement a linear support-vector classifier (SVC) that separates the data "as accurately as possible", using a separating hyperplane (in our case: a straight line, as we are considering the situation in the plane). What we want to achieve in the end is that the separating line's distance from the data should be as large as possible (indicated by a large so called "margin"), so that also test data can be classified as accurately as possible. As this usually isn't possible in general situations (e.g. when a lot of randomness is present in the data), certain variables, that don't exactly fit in this framework have to be "punished" using so called slack-variables. It might e.g. be allowed for a certain amount of points to lie inside the margin or lie on the wrong side of the borders. This procedure in many situations can improve the generalization performance of the classifier. In any case, we won't go into the details here, and just mention that the amount of this "punishment" is encoded in the hyperparameter $C$ for the SVC. For further information consult e.g. the lecture notes for the supervised learning course.

We provide the code for the implementation of the SVC using the sklearn-toolkit. Also a function, that outputs the margin for a SVM classifier is provided. For more details, consult https://scikit-learn.org/stable/modules/generated/sklearn.svm.SVC.html#sklearn.svm.SVC. Your task is now the following:

Task:

Write a function "compute_svm" that again depends on a parameter $k\in \mathbb{N} \geq 10$, and that computes linear SVCs for up to $k$ data points created via the "create_data" function using a for-loop. For each number of data points the corresponding margin should be printed and stored in a list. So in the end for each $10 \le j \le k$ a different SVC for $j$ data distributed according to the procedure from the previous question should be computed and the corresponding margin should be stored. The output of the function should be the list containing the margins. Run compute_svm(1000) and store the outputted list. The code for computing the SVC and its margin for a fixed number of data is provided.

Task:

Now we want to relate the behavior of the margin to theoretical results concerning the VC-dimension: It holds that for data which lie in a ball of radius $R$, the VC-dimension $d_{VC}$ of a linear support-vector classifier with margin $\gamma$ is bounded by $d_{VC} \le \frac{R^2}{\gamma^2}$ (cf. page 63 in the lecture notes). Using this fact, visualize the dependence of the margin and the VC dimension on the number of data points in two separate plots, using the previously produced list of margin values. Remember that all our data are contained in the unit circle.

def create_data(k):
    r = np.random.uniform(0, 1, size=k)
    phi = np.pi * np.random.uniform(0, 2, size=k)
    a = np.array([np.sqrt(r) * np.cos(phi)])
    b = np.array([np.sqrt(r) * np.sin(phi)])
    X = np.concatenate((a.T, b.T),axis=1)
    y = [+1 if elem>0 else -1 for elem in a.ravel() ]
    
    # plot the data with colored label
    if False: 
        color=['red' if elem2==+1 else 'blue' for elem2 in y ] #  ['red' if elem>0 else 'blue'  for elem in a.ravel() ]
        fig = plt.figure()

        ax.add_patch(circ)

    return X,y

def compute_svm(k):
    X_list = []
    y_list = []

It should be clearly visible from the plot, that, up to small fluctuations, the value of the margin decreases for a larger number of data points. This should come as no surprise, since the model gets increasingly complex for a larger number of data, and therefore a clearer separation is harder, which is indicated by smaller margins. Also the increase of the VC-dimension is reasonable, since VC-dimension provides an indicator for model complexity and it indeed seems to be a reasonable concept that can provide a theoretical basis for measuring the quality of widle applied classifier algorithms.
